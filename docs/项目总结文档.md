# 个人待办事项管理系统 — 项目总结文档

## 项目概述

本项目是一个基于 C++ 的命令行个人待办事项管理系统，支持任务的增删改查、状态管理、优先级管理、截止日期管理、标签分类、数据备份恢复和到期提醒。采用三层架构设计，以 CSV 文件作为持久化存储。

- **项目名称**：Personal To-Do List Management System
- **版本**：v2.0
- **语言标准**：C++11
- **开发环境**：Dev-C++ / MinGW GCC
- **数据存储**：CSV 文件（RFC 4180 兼容）

## 技术架构

### 三层架构

```
┌─────────────────────────────────────┐
│          表示层 (UI Layer)           │
│  main.cpp → ui.h/ui.cpp             │
│  主菜单 / 用户交互 / 彩色输出        │
├─────────────────────────────────────┤
│        业务逻辑层 (Business Layer)   │
│  task.h/task.cpp                    │
│  CRUD / 查询 / 排序 / 统计 / 标签    │
├─────────────────────────────────────┤
│         数据层 (Data Layer)          │
│  storage.h/storage.cpp              │
│  CSV 读写 / 备份 / 恢复              │
└─────────────────────────────────────┘
```

### 模块依赖关系

```
main.cpp
  └── ui.h ──┬── task.h  (Task struct, TaskStatus enum, 所有业务函数)
              │
              └── ui.cpp ──┬── storage.h ── storage.cpp
                           │
                           └── task.h ────── task.cpp
```

### 数据结构

```cpp
struct Task {
    int id;                 // 自动生成的唯一 ID
    std::string title;      // 标题（必填）
    std::string description;// 描述
    std::string dueDate;    // 截止日期（YYYY-MM-DD）
    int priority;           // 优先级（1-5，1=最高）
    TaskStatus status;      // Pending / Completed
    std::string tag;        // 标签/分类（v2.0 新增）
};
```

## 功能列表

### v1.0 功能
- 添加/删除/修改/查询待办事项
- 标记完成状态
- 优先级管理（1-5 级）
- 截止日期管理
- 标题关键字搜索（大小写不敏感）
- 按状态/优先级筛选
- 统计信息（总数/待完成/已完成）
- CSV 文件持久化
- CSV 特殊字符转义

### v2.0 新增功能
- **排序 UI**：按优先级/截止日期升序降序排序
- **标签分类**：任务标签字段，支持按标签搜索筛选
- **数据备份恢复**：一键备份到 `tasks_backup.csv`，支持从备份恢复
- **到期提醒**：启动时检查即将到期（3天内）和已逾期的任务
- **控制台美化**：ANSI 颜色、emoji 图标、Unicode 边框、优先级/状态颜色高亮
- **更丰富的统计**：完成百分比、全完成庆祝提示

## 技术亮点

### 1. CSV 特殊字符处理
实现了 RFC 4180 兼容的 CSV 转义：
- 包含逗号、引号、换行的字段自动用双引号包裹
- 字段内双引号自动转义为两个双引号
- 自定义解析器支持嵌套引号

### 2. 控制台美化
- 使用 ANSI escape codes 实现彩色输出
- Windows 下通过 `SetConsoleMode` 启用虚拟终端处理
- 优先级 1（红）、2（品红）、3（黄）、4（青）、5（绿）颜色编码
- Unicode box-drawing 字符绘制边框
- Emoji 图标增强视觉识别

### 3. 到期提醒算法
- 获取本地日期
- 字符串比较快速识别已逾期任务
- 日期差估算（3 日内）识别即将到期任务
- 启动时自动检查并显示提醒横幅

### 4. 模块化设计
- 头文件与实现分离
- 清晰的三层架构
- 每层职责单一，便于测试和维护
