# 个人待办事项管理系统小组分工安排

## 小组成员

本项目按 5 人小组进行分工，其中包括组长本人。

| 角色 | 成员 | 主要职责 | 交付内容 |
| --- | --- | --- | --- |
| 组长 / 项目统筹 | 成员 （刘阳） | 负责仓库创建、任务分配、进度协调、最终整合、会议记录组织 | README、分工安排、最终代码整合、提交检查 |
| 程序入口与菜单负责人 | 成员 （张旭润） | 负责主菜单、功能选择、输入输出交互 | `src/main.cpp`、`src/ui.cpp`、`include/ui.h` |
| 待办事项功能负责人 | 成员 （梁倚云） | 负责添加、删除、修改、查询、完成状态等核心业务逻辑 | `src/task.cpp`、`include/task.h` |
| 文件存储负责人 | 成员 （张宇航） | 负责 CSV 数据文件读写、数据保存和加载 | `src/storage.cpp`、`include/storage.h`、`data/tasks.csv` |
| 文档与测试负责人 | 成员 （贺平方） | 负责说明文档、用户手册、测试用例、截图整理 | `docs/项目说明文档.md`、测试记录、界面截图 |

## 推荐开发顺序

1. 组长刘阳创建 GitHub 仓库并上传基础结构。
2. 成员 张旭润 完成菜单框架，使程序可以运行。
3. 成员 梁倚云 完成待办事项的核心增删改查逻辑。
4. 成员 张宇航 完成数据文件读取和保存。
5. 成员 贺平方 同步编写说明文档、用户手册和测试记录。
6. 组长 刘阳 负责合并分支、解决冲突、统一代码风格并准备最终提交材料。

## 每个人需要遵守的规则

- 每个人只优先修改自己负责的文件，避免同时修改同一个文件导致冲突。
- 每完成一个小功能就提交一次，不要等全部写完再提交。
- 提交信息要写清楚，例如：`完成添加待办事项功能`。
- 修改别人负责的文件前，先在群里说明原因。
- 每次开始写代码前，先执行 `git pull` 获取最新版本。
