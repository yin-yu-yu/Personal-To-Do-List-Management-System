## 讨论内容

### 1. 项目整体目标

一致同意选择「个人待办事项管理系统」作为小组项目课题。该系统面向个人日常学习和生活管理场景，采用 C++ 控制台程序实现，数据通过本地 CSV 文件存储。

### 2. 系统主要功能

经讨论确定 1.0 版本功能范围：

- 添加待办事项（标题、描述、截止日期、优先级）
- 查看全部待办事项（表格形式展示）
- 修改待办事项
- 删除待办事项
- 标记待办事项为已完成
- 按标题/状态/优先级搜索待办事项
- 查看统计信息（总数、待完成数、已完成数）
- CSV 文件数据持久化保存和加载

2.0 版本预留：排序功能、到期提醒、UI 美化、分类标签。

### 3. 技术方案

- 开发语言：C++（C++11 标准）
- 数据存储：CSV 文本文件（兼容 RFC 4180）
- 开发工具：Dev-C++ IDE，备选命令行 MinGW GCC
- 版本控制：Git + GitHub，使用 GitHub Flow 分支策略
- 编码风格：统一使用 UTF-8 编码，中文注释

### 4. 模块架构

```
main.cpp → ui → storage + task
```

- `task` 模块：定义 Task 结构体和 TaskStatus 枚举，实现 CRUD、查询、排序、统计
- `storage` 模块：实现 CSV 文件读写，处理特殊字符转义
- `ui` 模块：控制台菜单、用户交互、展示格式化
- `main` 模块：程序入口，调用 ui 模块启动应用

### 5. 当前问题

1. 团队成员 Git 使用经验不统一 → 由组长编写 GitHub 协作指南
2. 成员开发环境可能不一致 → 统一使用 C++11 标准，避免平台特定代码
3. CSV 文件中可能存在逗号、引号等特殊字符 → storage 模块需要实现转义处理

## 会议结论

1. 项目正式确定为「个人待办事项管理系统」，采用 C++ + CSV 存储方案。
2. 分工明确，每人独立负责自己的模块文件，减少冲突。
3. 采用 GitHub Flow 分支策略：`feature/姓名-功能` → PR → `main`。
4. 开发顺序：菜单框架 → 业务逻辑 → 文件存储 → 文档测试 → 最终整合。
5. 约定每次提交前先 `git pull` 获取最新代码，避免大规模合并冲突。
6. 2.0 版本功能（排序、提醒、UI 美化）留待后续迭代。
